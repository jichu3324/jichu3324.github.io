MySQL高级学习

发表于 2022-07-07 更新于 2022-07-13 分类于 数据库
本文字数： 1.6k 阅读时长 ≈ 1 分钟

​ 本篇内容是在基础知识上的进阶，主要是表设计的相关内容以及多表查询方式，最后简单学习下事务的相关内容

约束

  • 约束是作用于表中列上的规则，用于限制加入表的数据

  • 约束的存在保证了数据库中数据的正确性、有效性和完整性

在MySQL中不支持检查约束

auto_increment 当列是数字类型且是唯一修饰，那么可以加该词修饰，当该列不给值得时候会自动添加并增长

非空约束

约束的列中不能出现NULL值

唯一约束

唯一性约束：受约束的列中不能出现重复的值

主键约束

作用相当于唯一约束和非空约束

受约束的列 非空且唯一

一张表中只有一个主键

默认约束

在赋值的时候如果没有给约束的列赋值，则会提供一个默认值

外键约束

外键用来让两个表的数据之间建立链接，保证数据的一致性和完整性

数据库设计

软件研发步骤：

数据库设计概念：

  • 数据库设计就是根据业务系统的具体需求，结合我们所选用的DBMS，为这个业务系统构造出最优的数据存储模型。

  • 建立数据库中的表结构以及表与表之间的关联关系的过程。

  • 有哪些表？表里有哪些字段？表和表之间有什么关系？

数据库设计的步骤：

  • 需求分析（数据是什么? 数据具有哪些属性? 数据与属性的特点是什么）

  • 逻辑分析（通过ER图对数据库进行逻辑建模，不需要考虑我们所选用的数据库管理系统）

  • 物理设计（根据数据库自身的特点把逻辑设计转换为物理设计）

  • 维护设计（1.对新的需求进行建表；2.表优化）

表关系：

  • 一对一

如：人和身份证

  • 一对多

如：部门和员工：一个部门对多个员工，一个员工对应一个部门

  • 多对多

如：商品和订单：一个商品对应多个订单，一个订单含有多个商品

表关系之一对多

实例：部门和员工

实现方式：在多的一方建立外键指向一的一方的主键

表关系之多对多

实例：商品和订单

实现方式：建立中间表，在该中间表中至少包括两个外键，分别关联两方的主键

表关系之一对一

实例：用户和用户详情表

实现方式：在任意一方加入外键，关联对方的主键，并且设计该外键为唯一

多表查询

根据特定的连接条件从多张表中查询到需要的数据

如果直接调用查询语句则会获得笛卡尔积。

笛卡尔积：

连接查询

内连接

查询两个表交集的数据

存在两种形式

  • 隐式内连接

在查询的过程中可以给表起别名，通过别名查询相关的资料

  • 显式内连接

外连接

左外连接

左侧表的所有数据和对应右侧表的符合条件的数据

如果右侧没有满足条件的数据的话，那么就为补null

右外连接

右侧表的所有数据和对应左侧表的符合条件的数据

如果左侧没有满足条件的数据的话，那么就为补null

子查询

子查询就是在查询中嵌套查询

嵌套的查询的结果分为三种：

  • 单行单列
  • 多行单列
  • 多行多列

单行单列

单行单列时，子查询的结果是一个值，因此可以在查询中当作一个数值使用

多行单列

多行单列时，查询后的结果是某个字段的多个值，在使用时可以用in来作为条件

多行多列

多行多列时，子查询后的结果相当于一个表格，因为该子查询可以作为一个表使用

同时还可以为该子查询形成的表起个别名

事务

数据库的事务是一种机制，一个操作序列，包括了一系列的数据库操作命令。事务把所有的命令作为一个整体一起向系统提交或撤销操作请求，即这一组数据库命令要么同时成功，要么同时失败。

事务是一个不可分割的工作逻辑单元

事务的四大特性

  • 原子性(A)：事务是最小单位，不可再分，要么同时成功，要么同时失败。
  • 一致性(C)：事务要求所有的DML语句操作的时候，必须使所有的数据都保持一致状态。
  • 隔离性(I)：事务A和事务B之间，操作的可见性
  • 持久性(D)：是事务的保证，事务终结的标志(内存的数据持久到硬盘文件中)，事务一旦提交或回滚，它对数据库中的数据的改变就是永久的。

MySQL中的事务默认是自动提交的

隔离性

隔离的四个级别：

  • 读未提交：read uncommitted
  • 读已提交：read committed
  • 可重复读：repeatable read
  • 串行化：serializable
