什么时面向对象

面向对象和面向过程

​ Java时一种面向对象的语言，所谓的面向对象，实际上是一种编程风格，除了面向对象，还有面向过程、指令式编程、函数式编程等，但是我们主要接触的只有前两种：面向对象编程和面向过程编程。

面向过程

​ 面向过程是一种以过程为中心的编程思想，是一种自顶向下的编程模式。

​ 在这个编程模式当中，问题被分解成一个个步骤，每个步骤用函数来实现，依次调用函数解决这个问题。

​ 面向过程的开发是具有流程化的，可以清晰的看到执行的步骤，明确知道每一步该做什么，代码的执行效率比较高。但是存在着复用性差、不易拓展、维护困难等问题

面向对象

​ 面向对象编程是将问题分解成一个个步骤，然后对每个步骤进行抽象，将相应的属性和方法封装字对象中，通过对象之间的组合调用解决问题。

​ 这种方式符合人们对现实世界的理解，同时编写的代码的可拓展性和可维护性更高

​ 面向对象编程具有三大基本特征：封装、继承和多态

面向对象的三大特征

封装

封装就是把客观事物封装成抽象的类，并且类的属性和方法只允许可信任的类和对象操作，对于不可信的类和对象则隐藏了相关的信息，防止程序中无关的部分意外的改变或者错误的使用了对象的私有部分。

继承

继承指的是类具有这样的能力，它可以使用现有类的所有属性和方法而不需要重新编写原来类的代码，同时还可以拓展具有自己的属性和方法。被继承的类被称为“父类”或者“基类”，而继承的类被称为“子类”或者“派生类”

继承的过程是从一般到特殊的过程

多态

多态是指同一个类实例的方法在不同的情况下有着不同的表现形式。在这种情况下，有着不同内部结构（具体操作，实现的业务逻辑）的对象可以共享同一个对外的接口。

常见的多态是将子类对象传入父类的引用中，当调用父类的方法的时候，通过传入的实际的子类来决定最终实现的具体的结构或者行为。
